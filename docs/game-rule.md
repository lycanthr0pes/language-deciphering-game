# ゲームルール仕様書

最終決定: @ly(らい) / PM
最終更新: 2026-07-20
ステータス: 仕様承認済み・実装未完了

## 1. 目的

一人称視点の暗号推理ゲームにおける、ゲーム進行、暗号規則、出題レベル、解答判定、誤答、時間制限、手帳、終了条件を定義する。

暗号フォントの詳細は`mende-kikakui-font-guide.md`、画面表現は`ui-spec.md`、実装責務は`implementation-spec.md`を正とする。

## 2. 基本ルール

- プレイヤーは、男が提示する暗号例文と日本語訳から暗号単語の意味を推理する。
- 男が喋るのは暗号例文、日本語訳、問題文だけとし、導入文と状況説明は地の文として表示する。
- Lv1からLv8まで順番に進み、Lv8正解後はクリア演出へ進む。
- 暗号例文は手帳へ記録し、次の問題へ進んでもリセットしない。
- プレイヤー向けUIへ内部カテゴリ名、仮英字、文字対応表を表示しない。

## 3. 日本語語彙

日本語語彙と内部カテゴリは確定仕様とする。

| 内部カテゴリID | 候補1 | 候補2 |
| --- | --- | --- |
| `color` | 赤い | 青い |
| `quality` | 大きな | 小さな |
| `humanNoun` | 男 | 女 |
| `animalNoun` | 犬 | 猫 |
| `quantity` | いくつかの | たくさんの |
| `verb` | 見る | 追う |

内部カテゴリIDは実装とテストだけに使用し、プレイヤーには見せない。

## 4. 暗号表記

### 4.1 表示方式

- 暗号は`Noto Sans Mende Kikakui`の実Unicode文字を使ったテキストとして表示する。
- Private Use Areaと仮英字は使用しない。
- 画像化せず、選択、折り返し、拡大ができるテキストとして扱う。
- 内部ロジックでは`color-1`のような`cipherId`を使い、画面では対応する`glyphText`だけを表示する。
- 正誤判定は字形ではなく、トークンIDと日本語正解データで行う。

### 4.2 推理構造

従来の「カテゴリを表す部分 + 候補番号を表す部分」という構造を、Mende Kikakuiの2文字で維持する。

- カテゴリ6種類へ1文字ずつ割り当てる。
- 候補1と候補2へ共通の文字を1文字ずつ割り当てる。
- 1暗号単語は`カテゴリ文字 + 候補文字`のコードポイント列で構成する。
- 使用する8文字は下表の現行割当とし、連続した範囲への再割当は行わない。
- 問題内の単語順は左から右、各暗号単語の内部は右から左に表示する。

| 用途 | コードポイント |
| --- | --- |
| 色 | `U+1E865` |
| 性質 | `U+1E822` |
| 数量 | `U+1E8A3` |
| 動詞 | `U+1E83D` |
| 人系名詞 | `U+1E845` |
| 動物系名詞 | `U+1E83A` |
| 候補1 | `U+1E854` |
| 候補2 | `U+1E827` |

仕様書内では表示環境に依存しないよう、暗号単語を`[color-1]`のような内部確認表記で示す。プレイヤー画面にはこの表記を出さない。

## 7. 出題データ

正解の日本語とトークン構成は確定仕様とする。暗号の画面表示は各`cipherId`から生成したMende文字列を使う。

| レベル | 内部確認用の問題 | 正解 |
| --- | --- | --- |
| Lv1 | `[color-2] [humanNoun-2]` | 青い 女 |
| Lv2 | `[color-1] [animalNoun-2]` | 赤い 猫 |
| Lv3 | `[quality-1] [animalNoun-1]` | 大きな 犬 |
| Lv4 | `[quantity-2] [animalNoun-2]` | たくさんの 猫 |
| Lv5 | `[humanNoun-1] [verb-2]` | 男 追う |
| Lv6 | `[color-1] [quality-2] [humanNoun-2]` | 赤い 小さな 女 |
| Lv7 | `[quantity-1] [humanNoun-1] [verb-1]` | いくつかの 男 見る |

同じ語彙、例文、問題、正解を複数ファイルへ重複定義せず、実装では単一のデータ定義から生成する。

## 8. 解答ルール

- 問題単独提示では従来の会話ボックスを使い、問題文をクリックして解答受付へ入った後は、問題文と解答UIを1つの統合ダイアログへ切り替える。
- 問題文の暗号単語を左クリックして解答対象を選ぶ。
- 選択中トークンの内部カテゴリに属する日本語候補2件だけを、カテゴリ名なしで表示する。
- 統合ダイアログでは各暗号単語と常設の枠付き解答欄を縦の組として並べ、日本語候補を左クリックすると選択した日本語を該当暗号単語の直下へ青色で表示する。
- 単語選択時には判定しない。
- 全暗号単語へ日本語を割り当てた後だけ`解答する`を有効にする。
- `解答する`を押した時だけ、問題全体をまとめて判定する。

## 9. 判定フィードバック

- 判定時に`正答 n / N`と、各日本語解答の`正答`／`誤答`を表示する。
- 正答はくすんだ緑`#67c587`、誤答はくすんだ赤`#c75b5b`を使い、枠または補助ラベルでも区別する。
- 判定結果を1400ms表示し、その間はタイマーと解答操作を止める。
- 判定中も統合解答ダイアログ、残り時間、`間違い可能 n`を表示したままにし、ステータス値のカウントだけを止める。
- 全問正答なら表示後に次レベルまたはクリア演出へ進む。
- 継続可能な誤答では、選択済み解答と判定結果を保持して解答へ戻る。
- 前回判定後に日本語解答を変更した時点で判定表示を消す。
- 前回判定から解答内容が変わっていない間は、同じ解答を再送信できない。

## 10. 誤答ルール

- 各問題開始時の`mistakesRemaining`は`1`。
- 1回目の誤答で`0`にするが、ゲームは続行する。
- `0`の状態でさらに誤答するとゲームオーバーを確定する。
- 次の問題へ進むと`1`へ戻す。
- `mistakeCount`は誤答のたびに1増やす。

## 11. 時間制限

- 時間制限を採用する。
- 変更可能な既定値は1問90秒、残り15秒から警告表示とする。
- 数値は`gameConfig.ts`へ集約する。
- 残り時間と`間違い可能 n`は問題単独提示中には表示せず、解答受付中と判定フィードバック中だけ右上へ表示する。
- タイマーは問題の解答受付中だけ進め、手帳を開いている間も停止しない。
- タイマーが0になった瞬間に終了時刻を保存し、判定フィードバックを挟まずゲームオーバー発砲演出へ進む。
- 時間切れは誤答ではないため、`mistakeCount`へ加算せず`wrong-answer.mp3`も鳴らさない。
- 正解して次の問題へ進むと時間を初期化する。
- 判定フィードバック、開始・終了演出中はタイマーを進めない。

## 12. 手帳

- 手帳には、提示済みの暗号例文と日本語訳、および正答済みの問題と解答を同じ形式で表示する。
- 現在レベルの例文は、問題文をクリックして解答受付へ入る瞬間に一括で履歴へ追加する。
- 正答した問題は、送信した日本語解答と組にして正答確定時に1件追加する。問題IDで重複を防ぎ、誤答と時間切れでは追加しない。
- 導入、例文、問題提示、判定中は開けず、解答受付中だけ操作できる。
- 推理を書き込む別メモやタブは設けない。
- 解答受付中に`Space`で開閉する。手帳内に閉じるボタンは置かない。
- 複数の見開きでは`A` / `D`で前後へ移動する。
- 履歴は時系列順に1見開き最大6件とし、左ページを3件まで埋めてから右ページを3件まで埋め、その後に次の見開きへ進む。
- 最初の見開きで`A`、最後の見開きで`D`を押しても範囲を越えない。
- 手帳を開いた直後は最新の見開きを表示する。
- 記録はレベルをまたいで保持し、正解後もリセットしない。
- Lv1〜Lv8をすべて正答した場合、提示例文10件と正答履歴8件の最大18件を保持する。
- `Tab`はゲーム操作に使わず、ブラウザ標準のフォーカス移動を維持する。

## 13. NEW通知

- 問題文をクリックして解答受付へ入り、新しい例文を一括保存した瞬間に`NEW`と下矢印を表示する。
- 通知全体を上下4px、1往復1800ms、`ease-in-out`で往復させる。
- `prefers-reduced-motion: reduce`では移動を止める。
- 通知後、閉じた状態の手帳を最初に開いた瞬間に既読とし、通知を消す。
- 同じレベルで閉じ直しても再表示しない。
- 次レベルで例文が追加された場合は新しい通知として表示する。
- 正答した問題と解答の追加では、`NEW`、未読状態、`write-note.mp3`を変更しない。
- リトライ時は既読状態を初期化する。

| 会話送り | 左クリック |
| 暗号単語選択 | 左クリック |

## 15. 開始・終了フロー

- 起動とリトライ時はCSSまばたき演出を行う。現行の背景・人物はCSS仮表示とし、将来画像を設定した場合だけ素材読込完了を待つ。
- 通常時の開始演出は2300ms、モーション低減時は300msとする。
- 演出中は会話、手帳、解答操作を受け付けない。
- Lv8正解時は男が銃を自分へ向けるクリア演出、ゲームオーバー時はプレイヤーへ向ける失敗演出を行う。
- 発砲・暗転後に、クリアでは`GAME CLEAR`、失敗では`GAME OVER`を表示する。
- 終了タイトル完了後にだけリザルトへ進む。
- 終了時刻はLv8の最終正解、終了条件となる誤答の送信時、または時間切れ確定時に1回だけ保存し、その後の判定表示と終了演出の時間を経過時間へ含めない。

## 16. 効果音

- 会話送り、男の発話、書き留め、銃を抜く、発砲の既存効果音を使う。
- `end.mp3`は`GAME CLEAR`／`GAME OVER`の表示開始時に1回だけ再生する。
- `close-note.mp3`は、開いている手帳を`Space`で閉じた瞬間に1回だけ再生する。
- `open-note.mp3`は、手帳を開いた時と、A/Dで見開きが実際に移動した時に1回だけ再生する。
- `wrong-answer.mp3`は、`解答する`で問題全体を判定して誤答だった時に1回だけ再生する。継続可能な誤答と終了条件となる誤答の両方を対象とし、時間切れでは鳴らさない。
- 全効果音を起動時に3要素ずつ非ブロッキングで先読みし、連続再生時は空き要素、なければ最古の再生要素を先頭から使う。
- 手帳を開く時、見開き移動時、NEW表示時には`close-note.mp3`を鳴らさない。

## 17. リザルトとリトライ

リザルトには次を表示する。

- 経過時間
- 正解回数
- 失敗回数
- 左クリックでリトライする案内

リトライ時は全ゲームstateを初期化し、開始演出から再開する。
