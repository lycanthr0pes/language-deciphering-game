# ゲームルール仕様書

最終決定: @ly(らい) / PM

## 1. 目的

一人称視点の暗号推理ゲームにおける、ゲーム進行、暗号ルール、出題レベル、解答判定、手帳の扱いを定義する。

この仕様書は、主に @かまぼこ(本物) と @ほっそー が実装時に参照するルール仕様である。UIや絵の見た目は `ui-spec.md` を参照する。

## 2. 基本ルール

- プレイヤーは暗号文と日本語訳の例文を見て、暗号単語と日本語単語の対応を推測する。
- 男は例文と問題のみを喋る。
- 導入文や状況説明は地の文として表示する。
- 例文と推測メモは次の問題に持ち越される。
- Lv1からLv8まで順番に進行する。
- Lv8の問題に正解するとクリア演出へ進む。

## 3. 確定事項と未確定事項

### 確定事項

- 日本語の単語は確定。
- 出題レベル構成は確定。
- 解答時は、品詞名・カテゴリ名をプレイヤーに見せない。
- 解答時は、選択中の暗号単語に対応する内部品詞・カテゴリ内の候補だけを表示する。
- 各問題で1回だけ誤答してもセーフ。

### 未確定事項

- 暗号表記は未確定。
- 存在しないアルファベット単語にするか、暗号フォントを作るかは未決定。
- この仕様書内の `raka` などの暗号単語は仮例として扱う。

## 4. 日本語語彙

日本語の単語は確定仕様とする。

| 内部品詞・カテゴリ | 候補1 | 候補2 |
| --- | --- | --- |
| 色 | 赤い | 青い |
| 性質 | 大きな | 小さな |
| 人系名詞 | 男 | 女 |
| 動物系名詞 | 犬 | 猫 |
| 数量 | いくつかの | たくさんの |
| 動詞 | 見る | 追う |

内部品詞・カテゴリ名は、実装上の分類として使う。プレイヤーには表示しない。

## 5. 暗号表記の仮ルール

暗号表記は仮例であり、最終的な見た目は未確定。

仮ルールでは、暗号単語を `語幹 + 語尾` で作る。

- 語幹は内部品詞・カテゴリを表す。
- 語尾は、その内部品詞・カテゴリ内の1つ目・2つ目の候補を表す。
- 語幹と語尾のセットで1単語として扱う。
- 語尾の対応は全カテゴリで共通にする。
- プレイヤーは例文から語幹と語尾のルールを推測する。

### 語幹の仮例

| 語幹 | 内部品詞・カテゴリ |
| --- | --- |
| `ra` | 色 |
| `do` | 性質 |
| `ta` | 数量 |
| `vi` | 動詞 |
| `hu` | 人系名詞 |
| `ke` | 動物系名詞 |

### 語尾の仮例

| 語尾 | 意味 |
| --- | --- |
| `ka` | 1つ目の候補 |
| `mi` | 2つ目の候補 |

### 暗号単語対応の仮例

日本語は確定。暗号単語は仮例。

| 内部品詞・カテゴリ | 1つ目 `ka` | 2つ目 `mi` |
| --- | --- | --- |
| 色 `ra` | `raka` = 赤い | `rami` = 青い |
| 性質 `do` | `doka` = 大きな | `domi` = 小さな |
| 数量 `ta` | `taka` = いくつかの | `tami` = たくさんの |
| 動詞 `vi` | `vika` = 見る | `vimi` = 追う |
| 人系名詞 `hu` | `huka` = 男 | `humi` = 女 |
| 動物系名詞 `ke` | `keka` = 犬 | `kemi` = 猫 |

## 9. 解答ルール

- 問題文の暗号単語を左クリックして解答対象を選ぶ。
- 選択中の暗号単語に対応する内部品詞・カテゴリ内の日本語候補だけを表示する。
- 品詞名・カテゴリ名は表示しない。
- 日本語候補は左クリックで選ぶ。
- 選択した日本語単語は、問題文の該当する暗号単語の下に表示する。
- 単語を選んだ時点では正解・不正解を判定しない。
- 全ての暗号単語に解答した後、`解答する` ボタンを押した時だけ合否判定する。
- 全ての暗号単語に解答していない場合、`解答する` ボタンは押せない。

## 10. 合否判定

- `解答する` ボタンを押した時点で、問題内の全単語をまとめて判定する。
- 全単語が正しい場合は正解。
- 1つでも間違っている場合は誤答。
- 正解した場合は、少し間を置いて次のレベルへ進む。
- Lv8で正解した場合はクリア演出へ進む。

## 11. 誤答ルール

- 各問題で1回だけ誤答してもセーフ。
- 各問題開始時の間違い可能回数は `1`。
- 誤答すると間違い可能回数を `1` 減らす。
- 間違い可能回数が `0` の状態でさらに誤答するとゲームオーバー。
- 次の問題へ進むと、間違い可能回数は `1` に戻る。

## 12. 手帳ルール

- 手帳は `Space` で開く。
- 手帳を開いている状態で `Space` を押すと閉じる。
- 手帳には `例文メモ` と `推測メモ` がある。
- `Tab` で例文メモと推測メモを切り替える。
- `A` / `D` キーで、現在開いているメモ内のページを切り替える。
- 手帳を開いた直後は、最後に追加されたページを表示する。
- メモを切り替えた直後は、そのメモの最後に追加されたページを表示する。

## 13. 例文メモ

- 男が提示した例文を記録する。
- 暗号例文と日本語訳の両方を記録する。
- 記録内容は次の問題に持ち越す。
- 正解後もリセットしない。

## 14. 推測メモ

- 暗号単語に対して、推測した日本語単語を結びつける。
- 表示形式は `暗号単語 → 日本語単語` とする。
- 品詞名・カテゴリ名は表示しない。
- 推測メモで日本語単語を選ぶときは、画面中央に単語リストを表示する。
- 単語リストは暗い背景パネルで表示する。
- 単語リスト上にも品詞名・カテゴリ名は表示しない。
- 推測メモの内容は次の問題に持ち越す。
- 正解後もリセットしない。

## 15. 操作ルール

| 操作 | 入力 |
| --- | --- |
| 会話送り | 左クリック |
| 解答対象の暗号単語選択 | 左クリック |
| 日本語候補の選択 | 左クリック |
| 解答する | 左クリック |
| 手帳を開く / 閉じる | `Space` |
| 例文メモ / 推測メモ切り替え | `Tab` |
| メモ内ページ移動 | `A` / `D` |
| リトライ | 左クリック |

## 16. クリア条件

- Lv8の問題に正解するとクリア。
- クリア後はクリア演出を再生する。
- クリア演出後にリザルト画面を表示する。

## 17. ゲームオーバー条件

- 間違い可能回数が `0` の状態で誤答するとゲームオーバー。
- 時間切れ後に誤答した場合もゲームオーバー。
- ゲームオーバー時は失敗演出を再生する。
- 失敗演出後にリザルト画面を表示する。

## 18. 時間制限

- 時間制限は採用する。
- 制限時間の数値はPMが最終決定する。
- 初期案は、1問あたり `90秒` とする。
- 初期案では、残り `15秒` から時間表示を赤にする。
- 時間切れだけでは即ゲームオーバーにしない。
- 時間切れ後に次に誤答した場合、ゲームオーバーにする。
- 正解して次の問題へ進むと、残り時間は初期値に戻る。

## 19. リザルト

リザルト画面では以下を表示する。

- クリア時間
- 正解回数
- 失敗回数

リトライは左クリックで行う。

## 20. 将来拡張

余裕があれば、以下を追加する。

- 日本語単語数を増やす。
- 暗号単語数を増やす。
- 特定の組み合わせのOK・NG設定を入れる。
- OK・NGの例から意味を推測させる問題を追加する。

例: `海 寝る` はNG。
