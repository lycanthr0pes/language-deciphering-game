# プロダクト全体フロー図

## 目的

一人称視点の暗号推理ゲームについて、ゲーム開始から終了までの進行、会話表示、暗号問題、正解時と不正解時の分岐を実装担当が理解できる形で整理する。

最終決定者はPMの@ly(らい)とする。

## 前提

- 実装はNext.jsの1ページ構成とする。
- サーバー処理や高度なルーティングは扱わない。
- Reactのstate、props、onClick、キーボード入力で進行を管理する。
- 操作は主に左クリック、Spaceキー、Tabキー、A/Dキーを使う。
- BGMは使わず、効果音のみ使用する。
- プレイヤーは椅子に縛られており、移動操作はない。
- ゲーム画面は一人称視点の固定画面とする。
- 男は例文と問題のみを喋る。
- 導入文や状況説明は地の文として表示する。
- Lv1からLv8まで順番に進行し、Lv8の問題に正解するとクリア演出へ進む。
- 各問題で1回だけ誤答してもセーフとする。

## 全体フロー

```mermaid
flowchart TD
    A[ゲーム起動] --> B[初期状態を生成]
    B --> C[開始演出: 暗転またはまばたき]
    C --> D[導入会話を表示]
    D --> E[現在Lvの暗号例文パート]
    E --> F[手帳に例文を記録]
    F --> G[問題提示]
    G --> H[解答受付]
    H --> I{解答は正解か}
    I -->|正解| J[正解回数を加算]
    J --> K{Lv8を正解したか}
    K -->|いいえ| L[次のLvへ進む]
    L --> E
    K -->|はい| M[クリア演出]
    M --> N[リザルト画面]
    I -->|不正解| O[失敗回数を加算]
    O --> P{判定前の間違い可能回数は0か}
    P -->|いいえ| S[間違い可能回数を1減らす]
    S --> H
    P -->|はい| Q[失敗演出]
    Q --> N
    N --> R{リトライ操作}
    R -->|左クリック| B
```

## ゲーム状態の流れ

実装では、画面全体の状態を`gamePhase`のようなstateで管理すると分かりやすい。

```mermaid
stateDiagram-v2
    [*] --> boot
    boot --> opening
    opening --> introDialogue
    introDialogue --> exampleDialogue
    exampleDialogue --> question
    question --> answering
    answering --> correctCheck
    correctCheck --> exampleDialogue: 正解かつLv1-7
    correctCheck --> clearCutscene: Lv8正解
    answering --> failureCheck: 不正解
    failureCheck --> answering: 間違い可能回数あり
    failureCheck --> gameOverCutscene: 間違い可能0でさらに不正解
    clearCutscene --> result
    gameOverCutscene --> result
    result --> boot: 左クリックでリトライ
```

## 会話表示フロー

会話は、通常会話、暗号例文、日本語訳、解答文で表示色を分ける。

| 種類 | 表示色 | 内容 |
| --- | --- | --- |
| 通常会話 | 白 | 導入文、状況説明 |
| 暗号例文 | 赤 | 男が提示する暗号文 |
| 日本語訳 | 青 | 暗号に対応する日本語訳 |
| 解答 | 青 | プレイヤーが選んだ日本語 |

```mermaid
flowchart TD
    A[会話パート開始] --> B[現在の会話行を取得]
    B --> C{会話行の種類}
    C -->|通常会話| D[白文字で表示]
    C -->|暗号例文| E[赤文字で表示]
    C -->|日本語訳| F[青文字で表示]
    C -->|解答| G[青文字で表示]
    D --> H[進行案内を表示]
    E --> H
    F --> H
    G --> H
    H --> I[左クリックを待つ]
    I --> J[会話送り効果音を再生]
    J --> K{次の会話行があるか}
    K -->|ある| B
    K -->|ない| L[次のゲーム状態へ進む]
```

導入会話は以下の順番で表示する。

1. ここは・・・？
2. 目を覚ますと、知らない場所にいた。
3. どうやら、椅子に縛られて動けないようだ。
4. 目の前の机には手帳とペンが置いてあり、その奥には仮面を付けた男が座っている。
5. 男が話しかけてきた・・・

導入会話は全て地の文として扱い、男のセリフにはしない。

## 暗号問題フロー

暗号例文パートでは、男が暗号文と日本語訳を交互に提示する。提示された内容はすべて手帳に記録される。

```mermaid
flowchart TD
    A[暗号例文パート開始] --> B[現在のLvを取得]
    B --> C[Lv定義から出題要素と追加例文数を取得]
    C --> D[日本語語彙と暗号単語対応を取得]
    D --> E[追加例文セットを生成]
    E --> F[暗号文を赤で表示]
    F --> G[会話送り待ち]
    G --> H[日本語訳を青で表示]
    H --> I[会話送り待ち]
    I --> J[手帳に例文を記録]
    J --> K{必要な例文をすべて提示したか}
    K -->|いいえ| F
    K -->|はい| L[Lv定義から問題文を取得または生成]
    L --> M[男が問題文を赤で提示]
    M --> N[手帳NEW表示と書き留め効果音]
    N --> O[暗号単語選択式の解答UIを表示]
```

## 手帳フロー

手帳はプレイヤーが過去の例文を見返し、暗号単語と日本語単語の対応を推測して記録するための補助UIとする。

```mermaid
flowchart TD
    A[通常画面] --> B{Space}
    B -->|入力あり| C[手帳を開く]
    C --> D[例文メモまたは推測メモを表示]
    D --> E{Tab}
    E -->|入力あり| F[例文メモと推測メモを切り替える]
    F --> D
    D --> G{A / D}
    G -->|入力あり| H[ページを移動する]
    H --> D
    D --> I{推測メモで暗号単語をクリック}
    I -->|クリックあり| J[カテゴリ名を伏せた中央単語リストを表示]
    J --> K{日本語候補をクリック}
    K -->|クリックあり| L[暗号単語に日本語を記録]
    L --> D
    D --> M{Space}
    M -->|入力あり| N[手帳を閉じる]
    N --> A
```

手帳の表示内容は以下を想定する。

| 項目 | 内容 |
| --- | --- |
| 例文履歴 | 男が提示した暗号文と日本語訳 |
| 推測メモ | `暗号単語 → 推測した日本語単語` の対応 |
| NEW表示 | 問題提示に切り替わった瞬間に出す通知。例文追加時や推測メモ更新時には出さない |

## 解答フロー

プレイヤーは日本語で解答する。入力欄に文字を打つのではなく、暗号単語をクリックし、日本語候補から対応を選んで答えを組み立てる。

```mermaid
flowchart TD
    A[問題提示] --> B[暗号単語を表示]
    B --> C{暗号単語をクリック}
    C -->|クリック| D[選択中の暗号単語を更新]
    D --> E[内部カテゴリ内の日本語候補だけ表示]
    E --> F{日本語候補をクリック}
    F -->|クリック| G[選択した日本語を暗号単語の下に表示]
    G --> H{全暗号単語が解答済みか}
    H -->|いいえ| B
    H -->|はい| I[解答するボタンを有効化]
    I --> J{解答するボタンをクリック}
    J -->|クリック| K[正解データと比較]
    K --> L{全暗号単語が一致しているか}
    L -->|はい| M[正解処理]
    L -->|いいえ| N[不正解処理]
```

解答UIでは、内部品詞・カテゴリ名はプレイヤーに表示しない。単語を選んだ時点では合否判定せず、全ての暗号単語に解答した後に`解答する`ボタンを押した時だけ判定する。

## 正解時の分岐

正解時は大きな演出を入れず、少し間を置いて次のLvへ進む。Lv8で正解した場合はクリア演出へ進む。

```mermaid
flowchart TD
    A[正解] --> B[正解回数を1増やす]
    B --> C[短い間を置く]
    C --> D{Lv8を正解したか}
    D -->|いいえ| E[次のLvへ更新]
    E --> F[間違い可能回数を1に戻す]
    F --> G[次の暗号例文パートへ]
    D -->|はい| I[クリア演出へ]
```

## 不正解時の分岐

各問題開始時の間違い可能回数は`1`とする。1回目の誤答はセーフで、同じ問題内でさらに誤答するとゲームオーバーになる。

```mermaid
flowchart TD
    A[不正解] --> B[失敗回数を1増やす]
    B --> C{判定前の間違い可能回数は0か}
    C -->|はい| D[ゲームオーバー確定]
    C -->|いいえ| E[間違い可能回数を1減らす]
    E --> F[解答を続行]
    D --> G[失敗演出へ]
```

## 失敗演出フロー

失敗時は紙芝居のように2シーン以上の切り替えで表現する。

```mermaid
flowchart TD
    A[失敗演出開始] --> B[男が銃を抜く]
    B --> C[銃を抜く効果音]
    C --> D[男が銃をこちらに向ける]
    D --> E[発砲]
    E --> F[発砲効果音]
    F --> G[暗転]
    G --> H[リザルト画面]
```

## クリア演出フロー

クリア時も銃を使うが、男が自分に向ける点が失敗演出と異なる。

```mermaid
flowchart TD
    A[クリア演出開始] --> B[男が銃を抜く]
    B --> C[銃を抜く効果音]
    C --> D[男が銃を自分に向ける]
    D --> E[発砲]
    E --> F[発砲効果音]
    F --> G[暗転]
    G --> H[リザルト画面]
```

## リザルトフロー

リザルト画面では、クリアとゲームオーバーのどちらでも結果を表示する。

## データ構造案

会話データは表示色を切り替えやすいように、テキストと種類をセットにする。

```ts
type DialogueLine = {
  speaker?: "narration" | "man" | "player";
  text: string;
  type: "normal" | "cipher" | "translation" | "answer";
};
```

暗号問題は、暗号単語ごとの対応を比較できる形にする。

```ts
type WordCategory =
  | "color"
  | "quality"
  | "humanNoun"
  | "animalNoun"
  | "quantity"
  | "verb";

type QuestionToken = {
  id: string;
  cipher: string;
  category: WordCategory;
  correctJa: string;
};

type Question = {
  level: number;
  tokens: QuestionToken[];
  correctAnswers: Record<string, string>;
  choiceCandidatesByTokenId: Record<string, string[]>;
};
```

`choiceCandidatesByTokenId`には、選択中の暗号単語に対応する内部カテゴリ内の候補だけを入れる。品詞名・カテゴリ名は画面に表示しない。

手帳の例文履歴は、暗号文と日本語訳をセットで保存する。

```ts
type ExampleRecord = {
  level: number;
  cipherText: string;
  translation: string;
  tokens: { cipher: string; ja: string; category: WordCategory }[];
};
```

## 暗号表記の仮ルール

暗号表記は未確定であり、以下は実装確認用の仮例とする。最終的に存在しないアルファベット単語にするか、暗号フォントを作るかはPMが決定する。

仮ルールでは、暗号単語を`語幹 + 語尾`で作る。

- 語幹は内部品詞・カテゴリを表す。
- 語尾は、その内部品詞・カテゴリ内の1つ目・2つ目の候補を表す。
- 語幹と語尾のセットで1単語として扱う。
- 語尾の対応は全カテゴリで共通にする。

| 語幹 | 内部品詞・カテゴリ |
| --- | --- |
| `ra` | 色 |
| `do` | 性質 |
| `ta` | 数量 |
| `vi` | 動詞 |
| `hu` | 人系名詞 |
| `ke` | 動物系名詞 |

| 語尾 | 意味 |
| --- | --- |
| `ka` | 1つ目の候補 |
| `mi` | 2つ目の候補 |

| 内部品詞・カテゴリ | 1つ目 `ka` | 2つ目 `mi` |
| --- | --- | --- |
| 色 `ra` | `raka` = 赤い | `rami` = 青い |
| 性質 `do` | `doka` = 大きな | `domi` = 小さな |
| 数量 `ta` | `taka` = いくつかの | `tami` = たくさんの |
| 動詞 `vi` | `vika` = 見る | `vimi` = 追う |
| 人系名詞 `hu` | `huka` = 男 | `humi` = 女 |
| 動物系名詞 `ke` | `keka` = 犬 | `kemi` = 猫 |

## PM決定項目

`game-rule.md`により、クリア条件、日本語語彙、Lv構成、誤答ルール、手帳操作、時間切れ後の扱いは確定済みとする。以下はPMが追加で判断する項目とする。

| 項目 | 決定内容 |
| --- | --- |

## 実装の優先順

1. 画面を1ページで表示する。
2. `gamePhase`と`currentLevel`で開始、会話、問題、リザルトを切り替える。
3. 左クリックで会話送りを実装する。
