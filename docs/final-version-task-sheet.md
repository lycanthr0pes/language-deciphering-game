# 完成版タスク管理用スプレッドシート設計

最終更新: 2026-07-21
ステータス: 完成版仕様案・PM承認待ち

## 1. この文書の扱い

この文書は、完成版実装の担当、依存関係、確認状況をGoogleスプレッドシートで管理するための入力データである。プロダクト仕様の正本ではない。

- 完成版仕様案: `final-version-spec-and-roles.md`
- 完成版サンプル: `final-version-sample-code.md`
- 現行仕様: `game-rule.md`、`mende-kikakui-font-guide.md`、`ui-spec.md`、`implementation-spec.md`、`project-flow.md`

PM承認前はタスクを`仕様確認中`とし、完成版コードの実装を開始しない。

## 2. スプレッドシートの列

次の1行をスプレッドシートの先頭へ貼り付ける。

```tsv
ID	カテゴリ	タスク	主担当	補助	依存	成果物	状態	確認方法	PM判定	PR
```

| 列 | 内容 |
| --- | --- |
| ID | `FV001`形式の固定ID |
| カテゴリ | PM、データ、UI、ゲーム進行、統合、検証 |
| タスク | 1つのPRで完了を判断できる作業 |
| 主担当 | 実装とPR作成の責任者 |
| 補助 | 仕様確認、レビュー、動作確認を行う担当 |
| 依存 | 先に完了または承認が必要なID |
| 成果物 | 完了時に存在するコード、Figma、確認記録 |
| 状態 | 入力規則から選択 |
| 確認方法 | 実装者が行う検証 |
| PM判定 | 未確認、確認中、修正依頼、承認 |
| PR | Pull RequestのURL |

## 4. 担当別一覧

PM以外の全員が実装を担当する。

| 担当 | 主担当タスク | コード実装 |
| --- | --- | --- |
| `@ささかまぼこ。` | FV006、FV007、FV008、FV009、FV015 | あり |
| `@かまぼこ(本物)` | FV010、FV011、FV012、FV013、FV016 | あり |
| `@ほっそー` | FV002、FV003、FV004、FV005、FV014 | あり |
| `@ly` | FV001、FV017 | なし。仕様承認と最終受入のみ |

### 編集競合を避けるルール

- `GameScreen.tsx`の主編集者は`@かまぼこ(本物)`とする。
- `@ささかまぼこ。`は子UIとCSS Modulesを完成させ、必要なcallbackを先に共有する。
- `@ほっそー`はデータ、共有型、設定値、純粋関数を完成させ、`GameScreen`側で必要な引数と返り値を先に共有する。
- 共有型の変更はFV002のPRを先にマージし、UIとゲーム進行が別々の型を定義しない。
- FV013までは、同じファイルを複数PRで同時編集しない。

## 5. 入力規則

スプレッドシートの入力規則へ使う。

```tsv
列	値	説明
状態	仕様確認中	PM承認前または仕様差分の確認中
状態	未着手	依存タスクを満たしているが作業未開始
状態	作業中	主担当が実装中
状態	レビュー待ち	PR提出済み
状態	修正中	レビュー結果へ対応中
状態	PM確認待ち	実装・レビュー・検証が完了
状態	完了	PM承認または担当範囲の受入完了
状態	保留	依存、素材、外部要因で停止
PM判定	未確認	PM未確認
PM判定	確認中	PM確認中
PM判定	修正依頼	修正が必要
PM判定	承認	PM承認済み
```

状態の更新条件:

- 実装だけでなく、行の`確認方法`まで完了してから`レビュー待ち`にする。
- 実行できなかった確認や既知の問題はPRとタスク行の両方へ記録する。
- 補助担当のレビューを終える前に`PM確認待ち`へ変更しない。
- PM以外の実装完了を、PMタスクFV001またはFV017の完了と混同しない。

## 7. PR前の共通確認

各実装PRで変更範囲に応じて実行する。

```bash
git diff --check
npm run lint
npm run build
```

UIまたはゲーム進行を変更した場合は`npm run dev`で実画面を確認し、スクリーンショットまたは動画をPRへ添付する。
