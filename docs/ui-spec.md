# UI・絵仕様書

担当: @ささかまぼこ。
最終決定: @ly(らい) / PM
最終更新: 2026-07-20
ステータス: 仕様承認済み・実装未完了

## 1. 目的

一人称視点の暗号推理ゲームに必要な画面レイアウト、UI、背景、キャラクター、モーション、素材制作の確定方針を定義する。

暗号文字は`mende-kikakui-font-guide.md`、ゲームルールは`game-rule.md`、実装責務は`implementation-spec.md`を正とする。

## 2. 全体方針

- 対象はPCブラウザ版とし、1920×1080の16:9を基準にブラウザ内で比率を維持して縮小する。
- プレイヤーは椅子に縛られた一人称視点で、移動操作はない。
- 中央奥に仮面の男、手前に机、机上に手帳とペン、下部に会話ボックス、右上に残り時間を配置する。
- 黒、こげ茶、暗い灰色、低彩度を中心に、暗く、不気味で、閉鎖的な雰囲気を優先する。
- 通常会話は白、暗号文は赤、日本語訳と判定前の解答は青を使う。
- 操作案内は会話・手帳などの本文領域外へ、小さな灰色で表示する。

## 3. 通常画面

```text
┌────────────────────────────────────┐
│                       残り時間 01:30 │
│                       間違い可能 1  │
│                                    │
│             仮面の男                │
│                                    │
│                机                  │
│         手帳              ペン      │
│                                    │
│ ┌──────────────────────────────┐ │
│ │ 会話・暗号・日本語訳           │ │
│ └──────────────────────────────┘ │
│ 左クリックで進む / Spaceで手帳    │
└────────────────────────────────────┘
```

### 背景と人物

- 窓のない小さな部屋とし、壁はコンクリート、古い木材、または汚れた漆喰を想定する。
- 画面端を黒く落とし、男の仮面、机、手帳だけがぼんやり見える照明にする。
- 男はスーツ姿、正面向き、椅子に座った状態を基本とする。
- 仮面と照明の最終案はPM判断のまま残す。
- 人物差分は通常、銃を抜く、プレイヤーへ向ける、自分へ向ける状態を用意する。

### 会話ボックス

- 位置は画面下部、幅は画面の80〜90%、高さは18〜25%を基準とする。
- 黒または濃い灰色、透明度70〜85%、細い灰色の枠、小さな角丸を使う。
- 男が喋るのは暗号例文、日本語訳、問題文だけとする。
- 導入文と状況説明は地の文とし、話者名を付けない。
- 男の名前や括弧は白または灰色、暗号本文は赤、日本語訳は青にする。

## 4. 暗号文字

- 暗号例文、問題文、選択対象トークン、手帳内暗号をすべて`Noto Sans Mende Kikakui`で表示する。
- 仮英字、Private Use Area、暗号画像は使用しない。
- Figmaとブラウザで同じTTF/OTF・WOFF2のソースとUnicode対応表を使う。
- 1単語はカテゴリ文字と候補文字の2文字で構成する。
- 問題内の単語順は左から右、単語内部は右から左にする。
- 2単語、3単語、Lv8の5単語で、境界と対応する日本語を追える余白を確保する。
- 通常会話、日本語訳、ボタン、リザルトへ暗号フォントを適用しない。
- 読み上げ用ラベルは`暗号単語1`のように順序だけを示し、意味を漏らさない。

## 5. 手帳

### 表示内容

- 手帳は通常画面上へ重ねるオーバーレイとし、背景を半透明の黒で暗くする。
- 表示するのは、提示済みの暗号例文と日本語訳の履歴だけとする。
- 見出しは`手帳`とする。
- 1ページに例文を2件表示する。
- ページ番号と前後へ移動できることを示すが、手帳内に閉じるボタンは置かない。

```text
┌──────────────────────────────┐
│             手帳              │
│                              │
│  暗号例文1                    │
│  日本語訳1                    │
│                              │
│  暗号例文2                    │
│  日本語訳2                    │
│                         2 / 3 │
└──────────────────────────────┘
Spaceで閉じる / A・Dでページ移動
```

### 操作

- `Space`で開閉する。
- 開いた直後は、最後に追加された例文を含むページを表示する。
- `A` / `D`で前後へ移動し、先頭・末尾の範囲を越えない。
- ページが1つの場合、ページ移動案内は非表示または無効表示にできる。
- `Tab`はゲーム操作に使わず、ブラウザ標準のフォーカス移動を維持する。

## 6. NEW通知

- 新しい例文を保存した瞬間ではなく、問題提示へ切り替わった瞬間に表示する。
- 手帳の位置を指す下矢印と`NEW`を一体の通知として表示する。
- 黒または暗い灰色を基本とし、くすんだ暗赤色を縁・影へ使用する。
- 通知全体を基準位置から上下4px動かす。1往復1800ms、`ease-in-out`、交互無限再生とする。
- `transform: translateY()`だけで動かし、レイアウト位置を連続変更しない。
- 通知自体はクリック対象にしない。
- 最初に手帳を開いた瞬間に既読となり、同じレベルでは再表示しない。
- 問題提示時に手帳が開いている場合は、最新ページへ移動して通知を出さない。
- `prefers-reduced-motion: reduce`では静止表示にする。

## 7. 解答UI

### 基本操作

- 男の問題文を赤い暗号文字で表示する。
- 暗号単語を左クリックすると、選択中トークンの内部カテゴリに属する日本語候補2件だけを表示する。
- プレイヤー向け画面にカテゴリ名、品詞見出し、カテゴリ別色分けを出さない。
- 日本語候補を左クリックすると、選択した日本語を暗号単語の下へ表示する。
- 全トークンを回答するまで`解答する`を無効表示にする。
- 合否判定は`解答する`を押した時だけ行う。

### 判定前

- 選択中トークンは青い枠または青い発光で示す。
- 選択済み日本語は青`#66aaff`、未選択は薄い横線または`未選択`とする。
- 日本語候補と解答ボタンは暗い背景、灰色の枠、青寄りの白文字とする。
- ホバー、フォーカス、無効状態を色だけでなく枠や明度でも示す。

### 判定後

- 解答UI上部またはボタン付近に`正答 n / N`を表示する。
- 正答した日本語はくすんだ緑`#67c587`、細い緑枠、`正答`ラベルを付ける。
- 誤答した日本語はくすんだ赤`#c75b5b`、細い赤枠、`誤答`ラベルを付ける。
- 暗号文自体は赤のままにする。
- 判定表示中は全操作を無効にする。
- 継続可能な誤答後も選択内容と判定表示を残し、日本語を変更した時だけ判定前表示へ戻す。
- Lv8の5単語でも、正答数と全トークンが同時に読めるよう折り返しを許可する。

## 8. 時間・間違い可能回数

- 残り時間を右上へ白文字で表示する。変更可能な既定値は90秒、残り15秒から赤にする。
- `間違い可能 n`を時間表示の下へ小さく常時表示する。
- 初期値は1、0になったら赤で示す。
- 判定フィードバック、手帳、開始・終了演出中は時間表示を維持したままカウントを止める。

## 9. 開始演出

- まばたき用画像は制作しない。
- 通常の背景、人物、机素材を読み込んだ後、実装側の黒い上下レイヤーでまぶたを描く。
- 通常時は次の2300msのタイムラインとする。

| 経過時間 | 視界 |
| --- | --- |
| 0〜300ms | 完全な暗転 |
| 300〜650ms | 細い隙間まで開く |
| 650〜820ms | 一度閉じる |
| 820〜1250ms | 半分程度まで開く |
| 1250〜1410ms | 短く閉じる |
| 1410〜2300ms | ゆっくり全開 |

- 演出中は入力を受け付けない。
- 素材読込失敗または5000msのタイムアウト時は、演出を始めずエラーと再読み込み案内を表示する。
- `prefers-reduced-motion: reduce`では、黒画面から300msで通常画面へフェードする。

## 10. 失敗・クリア演出

### 発砲まで

- 失敗時は、男が銃を抜き、プレイヤーへ向け、発砲して暗転する。
- クリア時は、男が銃を抜き、自分へ向け、発砲して暗転する。
- 紙芝居式の人物差分を使い、発砲フラッシュは1回、最大100msとする。

### 終了タイトル

- 発砲・暗転後、失敗時は`GAME OVER`、クリア時は`GAME CLEAR`をテキストで表示する。画像素材は作らない。
- `GAME OVER`は暗い赤で衝撃表示と小さな横揺れを行い、2300ms表示する。
- `GAME CLEAR`は象牙色と淡い金色で、浮上、字間収束、淡い光を行い、2400ms表示する。
- モーション低減時は1500msの単純なフェードにする。
- タイトル完了前にリザルトを表示しない。

## 11. リザルト

- 黒背景または暗い部屋をぼかした背景を使う。
- 見出しは終了タイトルと重複させず`RESULT`とする。
- 経過時間、正解回数、失敗回数を白で表示し、数値を少し大きくする。
- 画面下部に`左クリックでリトライ`を小さく表示する。

## 12. 音の表示タイミング

- 男の発話音は、暗号例文、日本語訳、問題文が表示された瞬間だけ鳴らす。
- 会話送り音は左クリックで会話を進めた時に鳴らす。
- 書き留め音は問題提示とNEW表示の開始時に鳴らす。
- 銃を抜く音と発砲音は対応する演出で鳴らす。
- `end.mp3`は終了タイトルの開始時に1回だけ鳴らす。
- `close-note.mp3`はSpaceで手帳を閉じた時に1回だけ鳴らす。
- `open-note.mp3`はSpaceで手帳を開いた時と、A/Dでページが実際に移動した時に1回だけ鳴らす。
- `wrong-answer.mp3`は誤答の判定表示を開始する時に1回だけ鳴らし、正解時には鳴らさない。

## 13. 制作物

### 必要なもの

- 1920×1080の通常、問題、手帳、判定、両終了タイトル、リザルト確認フレーム。
- 背景、通常人物、銃を抜く人物、プレイヤーへ向ける人物、自分へ向ける人物、机、手帳、ペン。
- NEW通知の静止デザイン。
- 判定前、正答、誤答の解答トークン差分。
- `GAME OVER`と`GAME CLEAR`の文字組みと最終静止状態。
- `Noto Sans Mende Kikakui`を適用した2単語、3単語、5単語の暗号フレーム。

### 不要なもの

- 手帳の別タブ、推理入力画面、中央候補リスト。
- 手帳の閉じるボタン。
- 仮英字版、Private Use Area版、暗号画像。
- 暗転、半目、全開のまばたき画像や黒マスク画像。
- NEW通知の位置差分画像。
- `GAME OVER`／`GAME CLEAR`の画像素材。
