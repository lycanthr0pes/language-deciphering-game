# UI・絵仕様書

担当: @ささかまぼこ。  
最終決定: @ly(らい) / PM

## 1. 目的

一人称視点の暗号推理ゲームに必要な、画面レイアウト・UI・背景・キャラクター・演出素材の制作方針をまとめる。

この仕様書は、@ささかまぼこ。 がFigmaで画面案と画像素材を作るための指示書である。最終的な見た目や素材の採用可否はPMが決定する。

## 2. 全体の画面方針

### 基本方針

- 画面は一人称視点。
- プレイヤーは椅子に縛られており、自由に移動できない。
- 画面中央奥に仮面の男、手前に木製の机を配置する。
- 暗く、不気味で、閉鎖的な雰囲気を最優先する。
- UIはゲーム画面の雰囲気を壊さないよう、暗色・低彩度でまとめる。
- 赤は暗号文、青は日本語訳・解答、白は通常会話に使う。

### 推奨画面比率

- 基本制作サイズ: `16:9`
- Figma推奨フレーム: `1920 x 1080`
- 実装時はブラウザ画面に合わせて縮小・拡大する想定。

## 3. 画面レイアウト

### 通常ゲーム画面

構成は以下を基本とする。

```text
┌────────────────────────────────────┐
│                            残り時間 │
│                                    │
│          暗い部屋・仮面の男          │
│                                    │
│              木製の机               │
│        手帳            ペン         │
│                                    │
│ ┌──────────────────────────────┐ │
│ │ 会話・暗号文・日本語訳表示欄   │ │
│ └──────────────────────────────┘ │
│ 左クリックで進む                 │
│ Spaceで手帳を開く                 │
└────────────────────────────────────┘
```

### 配置指定

- 右上: 残り時間表示。
- 中央奥: スーツ姿で仮面を付けた男。
- 中央手前: 質素な木製の机。
- 机上: 手帳とペン。
- 画面下部: 会話ボックス。
- 会話ボックス下: 操作案内テキスト。
- 男が喋るのは、暗号の例文提示と問題提示のみ。

### 優先度

- 最も目立つもの: 仮面の男。
- 次に目立つもの: 会話・暗号・日本語訳。
- 補助的に見せるもの: 手帳、ペン、残り時間。

## 4. 背景の雰囲気

### 部屋

- 窓のない小さな部屋。
- 壁はコンクリート、古い木材、または汚れた漆喰のイメージ。
- 色は黒、こげ茶、暗い灰色を中心にする。
- 奥行きは浅めにして、閉じ込められている感覚を出す。

### 照明

照明案は以下のどれかをPMが最終決定する。

- 案A: 天井の裸電球
- 案B: 卓上ランプ
- 案C: ろうそく

推奨は案Aの裸電球。理由は、仮面の男の顔だけを不気味に照らしやすく、監禁されている雰囲気が出るため。

### 光の演出

- 画面全体は暗め。
- 男の仮面、机、手帳だけがぼんやり見える。
- 画面端は黒く落とす。
- フェードイン・暗転に対応しやすいよう、背景は暗色ベースで作る。

## 5. 仮面の男

### 基本デザイン

- スーツ姿。
- 正面向き、椅子に座っている。
- 目の前のプレイヤーを見ている構図。
- 顔は狐の仮面、または木の仮面で隠す。

### 仮面案

- 案A: 狐の仮面
- 案B: 木の仮面

推奨は案Aの狐の仮面。理由は、シルエットで分かりやすく、ゲーム画面内で不気味さを出しやすいため。

### 必要な差分

- 通常: 座っている。
- 銃を抜く: 手元に銃が見える。
- 銃をこちらに向ける: ゲームオーバー用。
- 銃を自分に向ける: クリア演出用。

## 6. 会話ボックス

### 表示形式

会話は以下のどちらかで作る。

- 案A: 半透明の黒い会話ボックス
- 案B: 文字だけが暗闇に浮く形式

推奨は案Aの半透明ボックス。理由は、暗号文・日本語訳を安定して読みやすくできるため。

### デザイン指定

- 位置: 画面下部。
- 幅: 画面横幅の80〜90%。
- 高さ: 画面高さの18〜25%。
- 背景色: 黒、または濃い灰色。
- 透明度: 70〜85%。
- 枠線: 細い灰色、またはくすんだ白。
- 角丸: 小さめ。丸くしすぎない。
- 文字色: 通常会話は白。

### 操作案内

操作案内は、会話ボックスや手帳などのダイアログボックスの外に出す。

画面最下部、またはダイアログボックスのすぐ下に小さく表示する。

- `左クリックで進む`
- `Spaceで手帳を開く/閉じる`
- 手帳表示中: `Tabで例文メモ/推測メモを切り替え`
- 手帳表示中: `A / Dでページ移動`

操作案内は本文より小さく、暗めの白または灰色にする。

## 7. 暗号文と日本語訳の表示

### 色分け

- 通常会話: 白
- 暗号文: 赤
- 日本語訳・解答: 青

### セリフの扱い

- 男が喋るのは、暗号の例文提示と問題提示のみ。
- 導入文や状況説明は、男のセリフではなく地の文として表示する。
- 例文提示では、男が暗号文と日本語訳をプレイヤーに向かって喋っている構図にする。
- 問題提示では、男が暗号文だけを出題する構図にする。

### 効果音タイミング

- 男が喋る効果音は、暗号例文、日本語訳、問題文など、男のセリフ行が表示された瞬間に鳴らす。
- 導入文や状況説明の地の文では、男が喋る効果音を鳴らさない。
- 会話送り効果音とは別に扱い、同じクリックで会話送り音と男の喋る音が続けて鳴ってもよい。

### 表示例

```text
男「rami humi」
男「青い 女」
```

### デザイン指定

- 暗号文は赤で表示する。
- 日本語訳は青で表示する。
- 暗号文と日本語訳は、上下に分けて表示すると読みやすい。
- 暗号例文と日本語訳は、男がプレイヤーに向かって喋っている構図で表示する。
- 日本語訳の単語は確定仕様として扱う。
- 暗号文の文字列は仮例として扱う。
- 暗号表記は、存在しないアルファベット単語にするか、暗号フォントを作るか未確定。
- Figmaでは、存在しないアルファベット単語版と、暗号フォント風版のどちらにも対応できる余白・文字サイズにする。

### 推奨レイアウト

```text
男「rami humi」
男「青い 女」
```

「男」の名前表示や括弧は白または灰色、暗号本文だけ赤、日本語訳だけ青にする。

## 8. 日本語語彙・暗号例・レベル構成

この章の分類名は制作・実装用の内部名である。プレイヤー向けUIには、色、性質、数量、動詞、名詞などの分類名を表示しない。

日本語の単語、出題範囲、レベル構成は確定仕様とする。

暗号単語の表記は仮例とする。存在しないアルファベット単語にするか、暗号フォントを作るかは未確定のため、以下の `raka` などの暗号表記は制作・実装確認用の例として扱う。

### 暗号表記の仮例

- 仮例では、暗号単語を `語幹 + 語尾` で作る。
- 仮例では、語幹が内部品詞・カテゴリを表す。
- 仮例では、語尾がその内部品詞・カテゴリ内の1つ目・2つ目の候補を表す。
- 語頭と語尾のセットで1単語として扱う考え方は維持する。
- プレイヤーは例文記録と推測メモから、語幹と語尾のルールを推測する。
- 仮例では、語尾 `ka` / `mi` は全カテゴリで共通にし、未知の暗号単語でも語尾から1つ目・2つ目の候補を推測できるようにする。
- 既出の語幹と語尾の組み合わせから、未提示の単語の意味を判断する推理を入れる。

### 語幹の仮例

| 語幹 | 内部品詞・カテゴリ |
| --- | --- |
| `ra` | 色 |
| `do` | 性質 |
| `ta` | 数量 |
| `vi` | 動詞 |
| `hu` | 人系名詞 |
| `ke` | 動物系名詞 |

### 語尾の仮例

| 語尾 | 意味 |
| --- | --- |
| `ka` | 1つ目の候補 |
| `mi` | 2つ目の候補 |

### 単語対応

日本語の単語は確定。暗号表記は仮例。

| 内部品詞・カテゴリ | 1つ目 `ka` | 2つ目 `mi` |
| --- | --- | --- |
| 色 `ra` | `raka` = 赤い | `rami` = 青い |
| 性質 `do` | `doka` = 大きな | `domi` = 小さな |
| 数量 `ta` | `taka` = いくつかの | `tami` = たくさんの |
| 動詞 `vi` | `vika` = 見る | `vimi` = 追う |
| 人系名詞 `hu` | `huka` = 男 | `humi` = 女 |
| 動物系名詞 `ke` | `keka` = 犬 | `kemi` = 猫 |

### レベル別の出題範囲

Lv1からLv8まで順番に進行し、Lv8の問題に正解したらクリア演出へ進む。

| レベル | 出題に使う要素 | 正解例 |
| --- | --- | --- |
| Lv1 | 色2個 + 人系名詞2個 | 青い 女 |
| Lv2 | 色2個 + 名詞4個 | 赤い 猫 |
| Lv3 | 性質2個 + 名詞4個 | 大きな 犬 |
| Lv4 | 数量2個 + 名詞4個 | たくさんの 猫 |
| Lv5 | 動詞2個 + 名詞4個 | 男 追う |
| Lv6 | 色2個 + 性質2個 + 名詞4個 | 赤い 小さな 女 |
| Lv7 | 数量2個 + 動詞2個 + 名詞4個 | いくつかの 男 見る |
| Lv8 | 色2個 + 性質2個 + 数量2個 + 動詞2個 + 名詞4個 | たくさんの 青い 小さな 犬 追う |

### 例文追加数

| レベル | 追加内容 | 追加例文数 | 累計例文数 |
| --- | --- | --- | --- |
| Lv1 | 色 + 人系名詞 | 3個 | 3 |
| Lv2 | 動物系名詞を追加 | 1個 | 4 |
| Lv3 | 性質を追加 | 1個 | 5 |
| Lv4 | 数量を追加 | 1個 | 6 |
| Lv5 | 動詞を追加 | 1個 | 7 |
| Lv6 | 色 + 性質 + 名詞 | 1個 | 8 |
| Lv7 | 数量 + 名詞 + 動詞 | 1個 | 9 |
| Lv8 | 全部入り | 1個 | 10 |

### 出題例

| レベル | 暗号問題 | 正解 |
| --- | --- | --- |
| Lv1 | `rami humi` | 青い 女 |
| Lv2 | `raka kemi` | 赤い 猫 |
| Lv3 | `doka keka` | 大きな 犬 |
| Lv4 | `tami kemi` | たくさんの 猫 |
| Lv5 | `huka vimi` | 男 追う |
| Lv6 | `raka domi humi` | 赤い 小さな 女 |
| Lv7 | `taka huka vika` | いくつかの 男 見る |
| Lv8 | `tami rami domi keka vimi` | たくさんの 青い 小さな 犬 追う |

### UI上の扱い

- 解答時の選択肢は、選択中の暗号単語に対応する内部品詞・カテゴリ内の候補だけを出す。
- ただし、プレイヤーには品詞名・カテゴリ名を表示しない。
- 単語リストは、対応する候補だけをラベルなしで並べる。
- 推測メモにも品詞名・カテゴリ名は出さず、`暗号単語 → 日本語単語` の対応だけを表示する。
- 余裕があれば、最終的に単語数を増やす。
- 余裕があれば、特定の組み合わせのOK・NG設定を追加し、そこから推測させる。例: `海 寝る` はNG。

## 9. 手帳画面

### 表示方法

- `Space` で開く。
- 手帳を開いている状態で `Space` を押すと閉じる。
- 通常画面の上に重ねるオーバーレイ形式。
- 背景は少し暗くして、手帳を前面に出す。
- 手帳画面は「例文メモ」と「推測メモ」に分ける。
- `Tab` で例文メモと推測メモを切り替える。
- `A` / `D` キーで、現在開いているメモ内のページを切り替える。
- 手帳を開いた直後は、最後に追加されたページを表示する。
- 例文メモの内容は、次の問題にも持ち越される。
- 推測メモの `暗号単語 → 日本語単語` の対応も、次の問題にも持ち越される。
- 正解後も手帳の内容はリセットせず、過去の記録を見ながら次の問題を解けるようにする。

### レイアウト

例文メモ。

```text
┌──────────────────────────────┐
│        手帳: 例文メモ          │
│                              │
│  男「rami humi」                │
│  男「青い 女」                  │
│                              │
│  男「raka huka」                │
│  男「赤い 男」                  │
└──────────────────────────────┘
```

推測メモ。

```text
┌──────────────────────────────┐
│        手帳: 推測メモ          │
│                              │
│  rami   →  青い                │
│  humi   →  女                  │
│  raka   →  赤い                │
│  huka   →  男                  │
└──────────────────────────────┘
```

### 必要な要素

- 手帳の見開き画像、または紙のUI。
- 例文メモ: これまでに男が喋った暗号例文と日本語訳の一覧。
- 推測メモ: 暗号の単語に対して、日本語の単語を結びつける対応表。
- プレイヤー側から品詞の種類が分からないようにする。
- UI上では `名詞`、`動詞`、`形容詞` などの品詞ラベルを表示しない。
- 推測メモは品詞ごとに分けず、暗号単語と日本語単語の対応一覧として表示する。
- 各行は `暗号単語 → 推測した日本語単語` の形にする。
- 推測メモで対応する日本語単語を選択するときは、画面中央に単語リストを表示する。
- 推測メモ用の単語リストは暗い背景のパネルにし、背後の手帳画面も半透明の黒で少し暗くする。
- 推測メモ用の単語リストは、選択中の暗号単語に対応する内部品詞・カテゴリの候補だけを表示する。
- ただし、単語リスト上に内部品詞・カテゴリ名や品詞名は表示しない。
- 単語リストから日本語単語を左クリックすると、選択中の暗号単語に対応として記録される。
- 例文メモと推測メモに書いた対応は、問題が切り替わっても残る。
- 各メモはページが増える想定にする。
- `A` / `D` キーで現在開いているメモ内のページを切り替える。
- `Tab` で例文メモと推測メモを切り替える。
- メモを切り替えた直後は、そのメモの最後に追加されたページを表示する。
- 操作案内は手帳ダイアログの外に表示する。内容: `Spaceで閉じる / Tabでメモ切替 / A・Dでページ移動`

### NEW表示

問題が出されたとき、手帳の上に以下を表示する。

例文が追加された時点では表示せず、問題提示に切り替わった瞬間に表示する。同じタイミングで書き留める効果音を鳴らす。

- 下矢印
- `NEW`

デザインは赤、白、または黄色系で目立たせる。雰囲気を壊さないよう、派手すぎるポップな表現は避ける。

## 10. 解答UI・選択肢ボタン

### 基本仕様

- 問題文は、男がプレイヤーに向かって出題しているセリフとして表示する。
- 問題文の暗号単語を左クリックすると、画面に並ぶ日本語単語リストから解答を選べる。
- 日本語単語リストの選択も左クリックで行う。
- 選択肢の選択はすべて左クリックで行い、キーボード選択は使わない。
- 日本語単語リストは、選択中の暗号単語に対応する内部品詞・カテゴリの候補だけを表示する。
- 日本語単語リストには品詞ラベルを出さず、プレイヤー側から品詞の種類が分からないようにする。
- 単語を選ぶたびに、問題文の該当する暗号単語の下に、選択した日本語単語を表示する。
- 単語を選んだ時点では正解・不正解の判定をしない。
- 全ての暗号単語に日本語単語を選んだ後、`解答する` ボタンを左クリックした時だけ合否判定を行う。
- 全ての暗号単語に解答していない間は、`解答する` ボタンを押せない見た目にする。

### レイアウト案

```text
┌──────────────────────────────┐
│ 男「raka kemi」                 │
│                              │
│  raka       kemi              │
│  [ 赤い ]   [ 未選択 ]          │
│                              │
│ 日本語単語リスト               │
│ [ 犬 ] [ 猫 ]                 │
│                              │
│        [ 解答する ]            │
└──────────────────────────────┘
```

### デザイン指定

- 問題文の暗号単語は、クリックできることが分かるように薄い下線、枠、またはホバー時の明るさ変化を付ける。
- 暗号単語の下に表示する選択済み日本語単語は青で表示する。
- 未選択の単語下には、`未選択`、空欄、または薄い横線を表示する。
- 日本語単語リストのボタン背景は、濃い灰色、または黒に近い色。
- 日本語単語リストのボタン枠線は灰色。
- 日本語単語リストのボタン文字は、青寄りの白、または日本語訳用の青。
- ホバー時は少し明るくする。
- 選択中の暗号単語は、青い枠線、または青い発光で示す。
- `解答する` ボタンは、全て解答済みの時だけ有効に見えるデザインにする。
- `解答する` ボタンが押せない時は、暗い灰色で無効状態にする。
- 誤答判定時は、問題文全体、または `解答する` ボタンを一瞬赤く光らせる。

### 注意点

- 日本語単語リストは読みやすさを優先する。
- ボタン数が増えても崩れないよう、折り返しを想定する。
- 品詞ごとの見出し、色分け、区切り線は出さない。
- ボタン本文は大きめにする。
- どの暗号単語に対して日本語単語を選んでいるのか、必ず分かる見た目にする。
- 合否判定は `解答する` ボタンを押した時だけ行い、単語選択中には行わない。
- Lv8では最大5単語の問題になるため、暗号単語と選択済み日本語表示が崩れないようにする。
- 横幅が足りない場合は、暗号単語ごとのまとまりを保ったまま折り返す。

## 11. 時間表示・間違い可能回数表示

### 時間表示

- 位置: 画面右上。
- 通常時: 白文字。
- 残り時間が少ない時: 赤文字。
- 表示例: `残り時間 01:20`

### 間違い可能回数

間違えてもよい回数は数字で常時表示する。時間表示の下に小さく置く。

- 各問題の初期値は `1`。
- 各問題で1回だけ間違えてもセーフ。
- 次の問題に進んだら `1` に戻す。
- 誤答するたびに数字を減らす。
- `0` の状態でさらに間違えるとゲームオーバーになる。
- 表示例: `間違い可能 1`
- 色: 白または灰色。
- `0` になったら赤にする。

## 12. 開始演出

### 必要な画面

- 暗転画面。
- まばたき風の黒いマスク。
- フェードイン後の通常画面。

### 演出イメージ

1. 画面が真っ暗。
2. 少しだけ視界が開く。
3. また暗くなる。
4. ゆっくり部屋が見える。
5. 導入会話が始まる。

Figmaでは、開始演出用に「暗転」「半目」「視界が開いた状態」の3枚を用意する。

## 13. 失敗演出

### ゲームオーバー時

紙芝居のような切り替えで表現する。

必要な絵は最低2枚、できれば3枚。

- シーン1: 男が銃を抜く。
- シーン2: 男が銃をこちらに向ける。
- シーン3: 撃たれて暗転。

### 表現方針

- 大きなアニメーションは不要。
- 1枚絵の切り替えで怖さを出す。
- 銃口が画面手前に向く構図にする。

## 14. クリア演出

### クリア時

失敗演出と似た流れだが、銃の向きが異なる。

必要な絵は最低2枚、できれば3枚。

- シーン1: 男が銃を抜く。
- シーン2: 男が銃を自分に向ける。
- シーン3: 撃って暗転。

### 注意点

- クリア演出は派手にしすぎない。
- プレイヤーが「助かったのか、何が起きたのか」と感じる余韻を残す。

## 15. リザルト画面

- クリア時間
- 正解回数
- 失敗回数
- リトライ案内

### レイアウト

┌──────────────────────────────┐
│            RESULT             │
│                              │
│  クリア時間    05:42          │
│  正解回数      8              │
│  失敗回数      2              │
│                              │
│  左クリックでリトライ             │

### デザイン指定

- 背景は暗転後の黒、または暗い部屋をぼかしたもの。
- 文字は白中心。
- 数字は少し大きくする。
- リトライ案内は画面下部に小さく表示する。

## 18. PM確認ポイント

PMは以下を決定する。

- 照明を裸電球、卓上ランプ、ろうそくのどれにするか。
- 仮面を狐の仮面、木の仮面のどちらにするか。
- 暗号表記を存在しないアルファベット単語にするか、暗号フォントにするか。
- 会話表示を半透明ボックス、文字のみのどちらにするか。
- 手帳画面を紙の見開きにするか、UIパネルにするか。
- 演出絵を最低枚数で作るか、余裕を持って3枚ずつ作るか。

## 19. 推奨決定案

迷った場合は、以下を初期案として制作する。

- 照明: 天井の裸電球。
- 仮面: 狐の仮面。
- 会話表示: 半透明の黒い会話ボックス。
- 手帳: 古い紙の見開き。
- 間違い可能回数: 時間表示の下に小さく表示。各問題の初期値は1。
- 失敗演出: 3枚構成。
- クリア演出: 3枚構成。

この案をもとにFigmaで初回レイアウトを作成し、PM確認後に素材制作へ進む。
