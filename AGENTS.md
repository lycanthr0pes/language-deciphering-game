## プロダクトの確定仕様

- 一人称視点の固定画面による、Next.js App Routerの1ページゲーム。DB、サーバー処理、移動操作、高度なルーティングは不要。
- Lv1からLv8まで順番に進み、Lv8正解後はクリア演出、リザルトへ進む。
- 男が喋るのは暗号例文、日本語訳、問題文。導入文と状況説明は地の文として扱う。
- プレイヤー向けの会話、問題、手帳では話者名と`「」`を一切表示せず、本文だけを表示する。`DialogueLine.speaker`は発話音の制御にだけ使う。
- 表示色は通常会話が白、暗号文が赤、日本語訳とプレイヤーの解答が青。
- プレイヤー向けUIには、内部の品詞名・カテゴリ名を表示しない。
- 問題中の暗号単語を選ぶと、その単語の内部カテゴリに属する日本語候補だけを表示する。
- 単語選択時には判定しない。全暗号単語への割り当て後に `解答する` を押した時だけ、問題全体をまとめて判定する。未回答があればボタンは無効。
- 判定時は`正答 n / N`を表示し、単語別結果は正答を緑、誤答を赤の色分けだけで示す。正答は1400ms表示してタイマーと解答操作を止める。継続可能な1回目の誤答は選択と色分けを保持したまま即座に解答を再開し、同じ解答も再送信できる。2回目の誤答は判定表示を挟まずゲームオーバー演出へ直行する。
- 継続可能な1回目の誤答時だけ、問題・解答パネルを左右約10px、減衰付きで320ms揺らす。パネル外の候補、解答ボタン、判定数、操作案内は揺らさない。2回目の誤答では揺らさず、`prefers-reduced-motion`では1回目も揺らさず色分けだけを表示する。
- 各問題の `mistakesRemaining` は1から始まる。1回目の誤答で0になり、0の状態でさらに誤答するとゲームオーバー。次の問題では1に戻す。
- 残り時間と`間違い可能 n`は、問題単独提示中には表示せず、`answering`と`answerFeedback`の解答UIコンテナが表示されている間だけ右上へ表示する。判定中は表示値を維持したままカウントを止める。
- タイマーは`answering`中なら手帳を開いていても進める。0になった瞬間に`endedAt`を一度だけ保存し、手帳を閉じて判定表示を挟まず`gameOverCutscene`へ進む。時間切れは`mistakeCount`へ加算せず、誤答音も鳴らさない。正解して次の問題へ進むと時間を初期化する。
- 手帳は、提示済みの暗号例文と日本語訳に加え、正答した問題と解答を同じ表示形式で記録する。手帳内では見出し、発話者名、括弧を付けず、暗号文と日本語訳だけを表示する。操作案内は見開き画像の外へ置く。問題文をクリックして`answering`へ入る時に現在レベルの例文を一括記録し、正答確定時にその問題と解答を無通知で1件追加する。`answering`中だけ`Space`で開閉でき、1見開きは左3件、右3件の順で埋めてから次へ進む。`A` / `D`で見開きを移動し、開いた直後は最新とする。閉じるボタンとゲーム内の`Tab`操作は設けない。
- `answering`へ入って例文を記録した時に、右上の時間表示直下に手帳アイコンを置き、その上へ矢印なしの`NEW`通知を表示する。アイコンと通知は操作可能な`answering`かつ手帳を閉じている時だけ表示し、アイコンはクリック対象にしない。最初に手帳を開いた時点で既読にし、`NEW`は上下4px、1往復1800msで動かし、`prefers-reduced-motion`では静止させる。
- 暗号は`Noto Sans Mende Kikakui`の実Unicode文字で表示する。カテゴリ文字は`U+1E865`、`U+1E822`、`U+1E8A3`、`U+1E83D`、`U+1E845`、`U+1E83A`、候補文字は`U+1E854`、`U+1E827`を使い、単語順は左から右、単語内部は右から左にする。仮英字とPrivate Use Areaはプレイヤー画面へ出さない。フォント読込に失敗した場合は別表記へフォールバックせず、解答進行を停止してエラーを表示する。
- 新規タブの初回開始とゲーム内リトライ時はCSSによるまばたき演出を行う。同一タブのブラウザ再読込・開発時の再マウントでは開始演出を省き、素材読込後に導入会話へ直接進んで不要な全面暗転を挟まない。クリア／失敗の発砲演出後は`GAME CLEAR`／`GAME OVER`を表示してからリザルトへ進む。
- 読込、開始、会話、問題、解答、判定、手帳、発砲、終了タイトル、リザルトの全場面で、Figma node `13:66`から書き出した同一の`scene-ceiling-light.svg`を同じ配置で使う。照明をCSS図形や別素材へ置き換えない。まばたき、発砲フラッシュ、発砲直後の暗転が一時的に照明を覆うことは演出として許容する。
- 画面配置はFigmaを基準とするが、人物と机の実画像は`public/assets/images`の現行PNGを正本とする。`masked-man-aim-self.png`のローポリゴン調と暗い黒・えんじ色を画風基準とし、他の人物3差分は同じ服色、`desk-notebook-pen.png`は机、フクロウ、手帳、ペンを含めて同じ面構成と陰影へ統一する。実行時のCSSフィルターで差分ごとの色を補正しない。
- 終了タイトル開始時は`end.mp3`、Spaceで手帳を閉じた時は`close-note.mp3`、手帳を開いた時と見開きが移動した時は`open-note.mp3`、誤答確定時は`wrong-answer.mp3`を各1回再生する。誤答音は継続可能／終了条件の両方で鳴らし、時間切れ、正解、判定表示の再描画では鳴らさない。全音源は起動時に3要素ずつ先読みし、画像と音声のURLは`basePath`対応の共通関数で組み立てる。
- `endedAt`はLv8の最終正解、終了条件となる誤答の送信時、または時間切れ確定時に1回だけ保存し、判定表示、発砲、終了タイトルの時間を結果へ含めない。
- リザルトは黒地にFigma node `13:66`由来の照明だけを背景として表示し、人物、机、手帳、ビネットを出さない。経過時間、正解回数、失敗回数を表示し、左クリックで全ゲームstateを初期化してリトライする。

日本語語彙、Lv別構成、例文数、具体的な遷移は `docs/game-rule.md` を参照し、同じデータを複数箇所へ重複定義しない。

## 未決定・調整可能な仕様

- 1問90秒、残り15秒で警告は初期案。数値は `src/lib/gameConfig.ts` に集約し、PMが変更できるようにする。
- 仮面と背景の最終表現はPM決定事項。照明はFigma node `13:66`のSVGで確定し、全場面で共通利用する。現行版は画風統一済みの人物4差分と机・手帳・ペン、および開いた見開き背景をローカル素材として使い、部屋だけCSSで補完する。
- 配置済みの人物・机素材とMendeフォントはゲーム開始前に読み込み、失敗時はエラーを表示する。未配置の任意音源は無音でも進行できる実装を維持する。

## 実装構成と責務

- TypeScriptとCSS Modulesを基本とする。全体に必要なリセット等を除き、グローバルCSSへコンポーネント固有スタイルを増やさない。
- `src/app/page.tsx` は `GameScreen` を表示する薄いエントリーポイントにする。
- ゲーム進行に関わるstateと遷移は原則 `GameScreen` に集約する。子コンポーネントはpropsを描画し、操作をcallbackで親へ通知する。
- `SceneCeilingLight`はFigma照明の共通描画、`DialogueBox`は通常会話・例文・問題単独提示、`ChoiceList`は透明な解答UIコンテナと問題・解答欄だけを収める内側パネル、およびパネル外の候補、判定数、送信、案内、`Notebook`は提示例文・正答履歴、`NotebookIndicator`は解答中の手帳アイコンとNEW通知、`CipherText`は暗号文字、`TimerDisplay`は時間表示、`OpeningBlink`は開始演出、`CutsceneScreen`は発砲演出、`EndTitleScreen`は終了タイトル、`ResultScreen`は結果表示を担当する。
- 子コンポーネントで、親と同じ意味のstate、正誤判定、タイマー進行、フェーズ遷移、ゲーム初期化を重複して持たない。
- 共有型は `src/lib/gameTypes.ts`、調整値は `src/lib/gameConfig.ts`、純粋な正誤判定は `src/lib/judgeAnswer.ts` に置く。
- 画像・音声は`public/assets/images`、`public/assets/sounds`を使う。Mendeフォントは`src/assets/fonts`、ライセンスは`public/licenses`へ置き、`next/font/local`で読み込む。
- `useState`、`useEffect`、ブラウザイベント、`onClick` を使うコンポーネントには、必要な境界にだけ `"use client";` を付ける。

## React・入力処理の注意

- stateを直接変更しない。前の値に依存する更新には関数形式のsetterを使う。
- render中にランダム生成やstate更新をしない。問題データは問題開始時に一度生成してstateへ保存する。
- state更新直後に古いstateを参照する処理を避け、必要なら更新前の値から次の値を計算する。
- キーボードlistener、タイマー、演出用timeoutは `useEffect` のcleanupで解除する。
- ゲーム画面は`user-select: none`として、左クリックやドラッグで本文・暗号・UIラベルを範囲選択させない。`Space`は入力欄等の編集可能要素を除いて`keydown`と`keyup`のブラウザ既定動作を抑止し、フォーカス中のボタンからフォーカスを外して押下させず、`answering`中の手帳開閉だけに使う。`Tab`と`Enter`の標準操作は妨げない。
- 選択肢、手帳、解答ボタンなどの操作で、背景の会話送りが同時発火しないよう `stopPropagation()` とフェーズ条件を適切に使う。
- 会話中、解答中、手帳表示中、演出中、リザルト中のどの入力が有効かを明示し、無関係なフェーズで処理しない。
- 判定ロジックは副作用のない関数として保ち、UIコンポーネント内へ埋め込まない。

## UI方針

- 16:9を基準にしつつ、ブラウザサイズに追従させる。
- 暗い、不気味、閉鎖的な雰囲気を優先する。黒、こげ茶、暗い灰色、低彩度を中心にする。
- 中央奥に仮面の男、手前に机と手帳、通常会話は1920×1080基準の`x=96, y=824, w=1728, h=144`へ置く。`answering`と`answerFeedback`では、透明な解答UIコンテナ内で話者名と括弧を付けない問題文、暗号単語、その直下の解答枠だけを大枠へ収め、候補、判定、送信ボタン、操作案内を大枠外へ配置する。解答時のコンテナは`x=96, y=608, w=1728, h=440`、判定時は`x=96, y=590, w=1728, h=458`を基準とし、残り時間と間違い可能回数を右上へ表示する。日本語候補は共通の大枠を持たず、各候補を150×54px相当、`#111`背景、灰色2px枠の独立ボタンで表示する。
- 操作案内は会話や手帳の本文領域の外へ、小さめの灰色で表示する。
- `DialogueBox`の導入文、暗号例文、日本語訳、問題単独提示は中央揃えにする。会話操作案内は枠外の中央下、進行表示は枠内の右下へ分離し、会話送りで`next→`、問題単独提示から解答受付へ入る時は`answer→`とする。
- 解答受付中の操作案内は`Spaceで手帳を開く`だけを表示する。手帳履歴は紙色の半透明背景と読みやすい文字サイズを使い、背景の高さを内容に合わせて下側の空白を作らない。ページ番号は`a/b`形式の背景付き表示として、見開き画像と重ならない範囲で拡大し、画像の上側外へ置く。
- UIを変更する際は、マウス、`Space`、`A` / `D`、標準の`Tab`フォーカス、無効状態、Mende文字の可読性も確認する。
- Figmaの状態名・タイミング注釈と、発砲演出の説明用字幕は制作資料として扱い、プレイヤー画面へ表示しない。
- 全場面で`SceneCeilingLight`を経由してFigma node `13:66`由来の照明SVGを描画する。粒状フィルターは適用せず、形状と縦グラデーションだけを使う。1920×1080基準のnode枠を`x=612, y=-112.5, w=696, h=1020`とし、その中の画像をFigmaどおり左右7.4%、上1.78%、下25%のインセットで配置する。
- 男と椅子はFigmaの横位置とサイズを維持し、1920×1080基準で通常人物を`x=636, y=0, w=648, h=648`、発砲時の人物差分を`x=600, y=-48, w=720, h=720`へ配置する。
- 通常、抜銃、プレイヤー照準、自分照準の切り替えで服の黒とえんじ色が跳ねないこと、机・フクロウ・手帳・ペンが人物と同じローポリゴン調に見えることを確認する。

## 検証

変更範囲に応じて、少なくとも次を実行する。

```bash
npm run lint
npm run build
```

UIやゲーム進行を変えた場合は `npm run dev` で実画面も確認する。自動テスト用scriptは現在ないため、必要な手動確認を省略しない。

手動確認の要点:

- 左クリックで会話が正しいフェーズだけ進む。
- 左クリックやドラッグでゲーム内文字が範囲選択されず、フォーカス中のボタンへSpaceを押しても候補選択、解答送信、リトライが発火しない。TabとEnterの標準操作は維持される。
- 導入、例文、問題単独提示中は残り時間と間違い可能回数を表示せず、問題文クリック後の解答UIコンテナと同時に表示する。
- 導入・例文・問題提示・判定中は手帳を開けず、`answering`中だけ`Space`と手帳中の`A` / `D`が動く。`Tab`の標準フォーカス移動、ブラウザ既定動作、背景クリックへ不要な干渉をしない。
- 解答UIで各暗号単語の直下に対応する解答枠があり、問題文と解答欄だけが大枠内、候補、送信ボタン、案内、判定数が大枠外にある。日本語候補を選択でき、全回答後のみ送信でき、送信時だけ判定される。
- 正答の判定中は内外を分けた解答UIとステータスを表示したまま、操作とタイマーが止まる。継続可能な1回目の誤答では選択と色分けを保持して即時再開し、同一解答も再送信できる。2回目の誤答では判定表示を挟まずゲームオーバー演出へ直行する。
- 継続可能な誤答後に日本語を変更した場合は、その解答欄の色分けだけを解除する。変更していない解答欄の色分けと直前の`正答 n / N`は次の送信まで維持する。
- 継続可能な1回目の誤答だけ問題・解答パネルが320ms揺れ、パネル外の候補とボタンは揺れない。2回目は揺れずにゲームオーバー演出へ直行する。単語別結果は可視ラベルなしの緑・赤だけで表示され、モーション低減時は揺らさない。
- 1回目と2回目の誤答、時間切れ即終了が仕様どおり分岐する。
- 手帳を開いている間もタイマーが進み、0秒で手帳を閉じて失敗演出へ進む。
- 正答した問題と解答が発話者名・括弧・見出しなしの通常例文と同じ形式で重複なく追加され、履歴が左3件、右3件の順で埋まる。手帳操作案内は見開き画像と重ならない。
- 手帳の左右ページ枠が見開き画像の紙面中央に合い、各履歴がページ枠内で中央揃えになり、手帳アイコンとNEWが`answering`かつ手帳を閉じている時だけ時間表示の下へ出る。
- 会話送りに`next→`、問題単独提示に`answer→`が表示され、解答受付中の案内が`Spaceで手帳を開く`だけになっている。リザルト案内は`左クリックでリトライ`とする。
- Lv1からLv8、開始演出、クリア／失敗演出、終了タイトル、リザルト、リトライを通してstateが正しく初期化・保持される。
- 同一タブのブラウザ再読込では開始暗転が再生されず、ゲーム内リトライでは開始まばたきが再生される。
- 読込、開始、通常進行、手帳、両発砲演出、両終了タイトル、リザルトの全場面で同じFigma照明SVGと配置が維持される。
- プレイヤー画面に内部品詞・カテゴリ名、仮英字、PUA文字が露出しない。

実行できなかった確認や既知の未確認事項は、完了報告に明記する。

## Git・Pull Request運用

- ユーザーから明示されない限り、commit、push、Pull Request作成、ブランチ切り替えは行わない。
- チーム運用では `main` から `work/担当者-作業内容` を作り、`main` へ直接pushしない。長期の `develop` / `release` ブランチは作らない。
- commitメッセージは `add: ...`、`fix: ...`、`update: ...`、`docs: ...` の形式で、変更内容が分かる日本語にする。
- Pull Requestは機能単位にし、本文に「やったこと」「確認したこと」「PMに確認してほしいこと」を書く。UI・絵・演出変更にはスクリーンショットまたは動画を添付する。
- 作業ツリーにある依頼外の変更はユーザーのものとして保持し、勝手に整形、削除、巻き戻し、commitしない。
